No claude.ai, **não existe um "scanner de artefatos salvos" nativo** que você possa referenciar diretamente por prompt entre conversas diferentes. Mas há formas de trabalhar com isso:

---

**Como artefatos funcionam hoje:**

Claude salva cada versão do artefato dentro da mesma conversa — você pode navegar entre versões pelas setas na parte inferior da janela do artefato. Mas ao abrir uma nova conversa, esse artefato não é acessível automaticamente.

---

**A solução certa: usar Projetos**

Projetos são workspaces salvos onde você faz upload de arquivos e instruções uma única vez. Todo novo chat dentro desse projeto começa com o contexto completo.

Para reutilizar artefatos entre conversas:
1. Crie um **Projeto** no claude.ai
2. Exporte o artefato (copie o código/conteúdo)
3. Faça upload como arquivo no Projeto
4. Em qualquer nova conversa dentro do Projeto, Claude terá acesso ao conteúdo

---

**Para referenciar um artefato na mesma conversa:**

Basta pedir diretamente no prompt, por exemplo:
- *"Atualize o artefato anterior com..."*
- *"Baseado no componente que você criou acima, adicione..."*
- *"Refatore o código do artefato para incluir..."*

Artefatos são ideais para conteúdo significativo e autocontido que você provavelmente vai querer editar, iterar ou reutilizar fora da conversa.

---

Se quiser, posso te ajudar a estruturar um Projeto para organizar e reutilizar seus artefatos de forma eficiente.

    17	- **Differentiation**: What makes this UNFORGETTABLE? What's the one thing someone will remember?

    27	## Frontend Aesthetics Guidelines
    28	
    29	Focus on:
    30	- **Typography**: Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial and Inter; opt instead for distinctive choices that elevate the frontend's aesthetics; unexpected, characterful font choices. Pair a distinctive display font with a refined body font.
    31	- **Color & Theme**: Commit to a cohesive aesthetic. Use CSS variables for consistency. Dominant colors with sharp accents outperform timid, evenly-distributed palettes.
    32	- **Motion**: Use animations for effects and micro-interactions. Prioritize CSS-only solutions for HTML. Use Motion library for React when available.     34	- **Backgrounds & Visual Details**: Create atmosphere and depth rather than defaulting to solid colors. Add contextual effects and textures that match the overall aesthetic.

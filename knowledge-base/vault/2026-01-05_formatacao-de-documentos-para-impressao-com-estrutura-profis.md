**PROMPT AVANÇADO REFINADO PARA OPUS 4.5:**

"Você receberá três arquivos extensos contendo informações diversas que precisam ser transformadas em um documento profissional de referência. Sua tarefa é realizar uma análise profunda do conteúdo, identificar padrões temáticos, estruturar o conhecimento de forma lógica e criar um material em formato Markdown pronto para exportação e impressão.

**ESPECIFICAÇÃO TÉCNICA CRÍTICA:**

O documento final deve ser entregue como um arquivo Markdown puro (.md), não como artefato HTML ou React. O Markdown deve utilizar sintaxe CommonMark padrão com extensões GitHub Flavored Markdown (GFM) quando apropriado, pois isso garante compatibilidade máxima com conversores para PDF e processadores de documentos como Pandoc, Marked, ou plataformas de publicação que suportam Markdown.

**ESTRUTURA DO ARQUIVO MARKDOWN:**

O arquivo deve começar com metadados em formato YAML front matter que facilitam o processamento e conversão. Esses metadados devem incluir título do documento, autor ou organização responsável, data de criação, versão do documento e outras informações relevantes que apareçam na capa quando o arquivo for convertido para PDF.

A hierarquia de títulos deve seguir a convenção Markdown padrão, usando hashtags para indicar níveis. O título principal do documento usa um único hashtag, capítulos principais usam dois hashtags, seções dentro de capítulos usam três hashtags, e assim sucessivamente até o sexto nível se necessário. Esta hierarquia semântica é crucial porque conversores de PDF a utilizam para gerar índices automáticos e marcadores de navegação.

**FASE 1 - ANÁLISE E MAPEAMENTO COGNITIVO:**

Antes de começar a formatação, conduza uma análise exploratória completa dos três arquivos que serão fornecidos. Identifique a natureza fundamental do conteúdo em cada arquivo, reconheça temas recorrentes que aparecem em múltiplos documentos, detecte relações conceituais entre diferentes seções mesmo quando elas não estejam explicitamente conectadas, e mapeie a hierarquia natural das informações apresentadas.

Procure entender qual é o propósito subjacente desse conjunto de informações e quem seria o público-alvo mais provável para este material consolidado, pois essas percepções influenciarão decisões estratégicas sobre profundidade de explanação, escolha de linguagem técnica versus acessível, e estrutura organizacional mais adequada.

**FASE 2 - ARQUITETURA INFORMACIONAL:**

Com base na sua análise inicial, desenvolva uma taxonomia clara e lógica dos temas principais identificados. Organize o conteúdo seguindo uma progressão que facilite tanto a leitura sequencial quanto a consulta pontual de tópicos específicos. Você pode considerar diferentes critérios de organização, como agrupamentos por similaridade conceitual, sequência temporal se o conteúdo tiver componente histórico ou processual, hierarquia de importância estratégica para o objetivo presumido do documento, ou fluxo didático que construa conhecimento progressivamente do básico ao avançado.

Crie uma estrutura clara de capítulos e subcapítulos que reflita essa organização intencional, garantindo que cada seção tenha um propósito definido e contribua para a compreensão geral. Cada capítulo deve funcionar como uma unidade coesa de conhecimento, mas também se integrar naturalmente ao todo.

**FASE 3 - ELEMENTOS ESTRUTURAIS EM MARKDOWN:**

Construa o documento seguindo padrões profissionais de publicação técnica, implementando os seguintes componentes usando sintaxe Markdown apropriada.

A página de capa deve ser criada usando um título principal de primeiro nível seguido de parágrafos centralizados contendo informações complementares. Como Markdown puro não suporta centralização nativamente, você pode usar comentários HTML inline apenas quando absolutamente necessário para elementos de capa, mantendo o restante do documento em Markdown puro. A capa deve apresentar o título principal, subtítulo explicativo se aplicável, indicação de data ou versão, e qualquer outra informação contextual relevante.

O índice pode ser gerado de duas formas. A primeira opção é criar um índice manual usando listas aninhadas com links internos do Markdown que apontam para âncoras nos títulos das seções. A segunda opção, mais simples, é deixar um marcador de posição indicando que o índice será gerado automaticamente pelo conversor de PDF, já que a maioria dos processadores modernos cria índices clicáveis baseados na hierarquia de títulos. Recomendo esta segunda abordagem por ser mais sustentável quando o documento passar por revisões.

**FASE 4 - FORMATAÇÃO E ELEMENTOS VISUAIS:**

Implemente recursos visuais que aumentem a legibilidade e facilitem a navegação usando as capacidades nativas do Markdown enriquecido.

Para tabelas de dados estruturados, utilize a sintaxe de tabelas do GitHub Flavored Markdown, que usa pipes verticais para delimitar colunas e hífens para separar cabeçalhos do corpo da tabela. As tabelas devem ter cabeçalhos claros e descritivos, alinhamento apropriado para o tipo de dado em cada coluna, e legenda explicativa logo abaixo quando a tabela contiver informações que necessitem contextualização adicional.

Para destacar informações importantes, insights estratégicos ou alertas, utilize blockquotes do Markdown, que são criados com o símbolo maior que no início da linha. Você pode criar diferentes tipos de caixas de destaque usando emojis ou indicadores textuais no início do blockquote, como usar o emoji de lâmpada seguido de texto em negrito para insights, triângulo de aviso para alertas críticos, ou estrela para destaques principais. Este approach mantém o documento em Markdown puro enquanto ainda cria diferenciação visual.

Para listas de itens relacionados, utilize listas não ordenadas com asteriscos ou hífens, e listas ordenadas com números seguidos de ponto quando a sequência for relevante. Listas aninhadas devem usar indentação de dois ou quatro espaços para criar hierarquia visual clara.

**FASE 5 - QUEBRAS DE PÁGINA E OTIMIZAÇÃO PARA IMPRESSÃO:**

Para controlar quebras de página no documento final impresso, insira marcadores de quebra de página usando comentários HTML que são reconhecidos pela maioria dos conversores PDF. O código específico é inserir a linha `<div style="page-break-after: always;"></div>` ou simplesmente `\newpage` se você estiver usando Pandoc como conversor, pois ele reconhece este comando LaTeX dentro de Markdown.

Posicione estas quebras estrategicamente antes de cada capítulo principal para garantir que novos capítulos sempre comecem em página nova. Também as utilize antes de tabelas extensas ou diagramas importantes para evitar que sejam cortados no meio. Evite quebras dentro de listas longas ou parágrafos conceituais que formem uma unidade de pensamento, pois isso prejudica a compreensão durante a leitura.

**FASE 6 - REFINAMENTO DE CONTEÚDO:**

Não se limite a reorganizar mecanicamente o conteúdo existente. Adicione valor através de aprimoramentos editoriais estratégicos. Isto pode incluir a criação de parágrafos introdutórios para cada capítulo que estabeleçam contexto e orientem o leitor sobre o que será abordado, frases de transição entre seções que criem fluidez narrativa, sumários executivos no final de capítulos particularmente densos que consolidem os pontos principais, e sínteses finais que conectem os aprendizados de diferentes partes do documento.

Se termos técnicos específicos aparecerem com frequência, considere criar uma seção de glossário ao final do documento, organizada alfabeticamente, onde cada termo receba uma definição clara e contextualizada.

**CÓDIGO E SKILLS PARA CONVERSÃO:**

Quando você fornecer o arquivo Markdown final, inclua também instruções práticas de como o usuário pode converter este arquivo para PDF pronto para impressão. As principais opções que você deve documentar são as seguintes.

Para conversão usando Pandoc, que é a ferramenta mais poderosa e flexível, o comando básico seria `pandoc documento.md -o documento.pdf`, mas para resultados profissionais o comando deve incluir parâmetros adicionais como `pandoc documento.md -o documento.pdf --pdf-engine=xelatex --toc --toc-depth=3 -V geometry:margin=2.5cm -V fontsize=11pt -V linkcolor=blue`. Este comando usa o motor XeLaTeX para renderização superior, gera índice automático até três níveis de profundidade, define margens confortáveis de dois centímetros e meio, estabelece tamanho de fonte legível, e mantém links clicáveis em azul.

Para usuários que preferem interfaces gráficas, aplicativos como Typora, Mark Text ou Visual Studio Code com extensões apropriadas permitem exportar Markdown para PDF diretamente através de menus, sem necessidade de linha de comando. Estes aplicativos aplicam estilos padrão que geralmente produzem resultados satisfatórios para documentos profissionais.

Para integração em fluxos de trabalho corporativos, plataformas como GitBook, Docusaurus ou ferramentas de documentação técnica podem importar o Markdown e gerar PDFs com branding personalizado e estilos corporativos predefinidos.

**SKILLS NECESSÁRIAS PARA ESTA TAREFA:**

Esta solicitação complexa requer que você aplique múltiplas capacidades cognitivas de alto nível que o Opus 4.5 possui em abundância. Você precisará de habilidades analíticas profundas para compreender conteúdo diverso e identificar estruturas conceituais latentes, competência em design informacional para criar taxonomias que façam sentido intuitivo, expertise em comunicação técnica para expressar ideias complexas com clareza, conhecimento detalhado de sintaxe Markdown e suas extensões, compreensão de princípios tipográficos e de legibilidade que se aplicam a documentos impressos, e capacidade editorial para refinar e aprimorar conteúdo mantendo fidelidade às informações originais.

**ENTREGÁVEL FINAL:**

Entregue o documento completo em um único bloco de código Markdown, claramente delimitado com três backticks e indicador de linguagem markdown no início. O resultado deve ser um arquivo que possa ser copiado diretamente para um editor de texto, salvo com extensão .md, e então convertido para PDF usando qualquer uma das ferramentas mencionadas, resultando em um documento profissional pronto para impressão e distribuição.

Após processar os três arquivos que serão fornecidos, você pode apresentar primeiro um resumo executivo em linguagem natural explicando a estrutura organizacional que você propõe, as principais decisões de arquitetura informacional, e a lógica por trás do agrupamento de conteúdos. Isso me permitirá validar sua abordagem antes que você gere o documento completo, ou você pode proceder diretamente se sentir que a estrutura é claramente superior e autoexplicativa.

Aguardo os três arquivos para processamento."

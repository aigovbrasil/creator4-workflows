WORKFLOW DE BACKUP INTELIGENTE E VERSIONADO PARA GITHUB
═══════════════════════════════════════

📌 Categoria: DevOps/Automação  
🏷️ Tags: #git #github #backup #versionamento #automação #shell  
📅 Data: 2025-01-05  
⚡ Complexidade: Intermediário  

───────────────────────────────────────
📖 DESCRIÇÃO:
───────────────────────────────────────
Sistema de backup automático e versionado usando Git/GitHub como espinha dorsal. Inclui compressão seletiva de ativos binários, estrutura de diretórios otimizada e automação via shell scripts. Aplicável ao projeto Leonardo OS e seus agentes.

───────────────────────────────────────
🔧 DEPENDÊNCIAS:
───────────────────────────────────────
- Git (versionamento)
- GitHub CLI (`gh`) (criação de repositórios remotos)
- ImageMagick (opcional - otimização de imagens)
- tar/gzip (compressão de arquivos)
- cron (agendamento - opcional)

───────────────────────────────────────
📁 ESTRUTURA DE DIRETÓRIOS:
───────────────────────────────────────

```
/backup_agentes/
├── agente_nome/
│   ├── code/          # Código-fonte (.py, .js, .sh)
│   ├── prompts/       # Prompts e blueprints (.md, .txt)
│   ├── assets/        # Imagens, vídeos, modelos 3D
│   └── docs/          # Documentação técnica
```

**Justificativa:**
- `code/` - Versionamento granular, fácil revisão
- `prompts/` - Ativos de PI de fácil leitura
- `assets/` - Arquivos binários com tratamento específico
- `docs/` - Documentação junto ao projeto

───────────────────────────────────────
💡 COMANDOS DE INICIALIZAÇÃO:
───────────────────────────────────────

```bash
# 1. Criar diretório e inicializar Git
mkdir ~/backup_agentes
cd ~/backup_agentes
git init

# 2. Criar repositório remoto no GitHub
# (requer autenticação prévia: gh auth login)
gh repo create nome-do-repo --public --source=. --remote=origin

# 3. Criar estrutura de diretórios (exemplo: Agente Jervis)
mkdir -p jervis/{code,prompts,assets,docs}
```

───────────────────────────────────────
⚙️ OTIMIZAÇÃO E COMPRESSÃO:
───────────────────────────────────────

### A. Otimização de Imagens (opcional)
```bash
# Compressão com ImageMagick (se disponível)
convert assets/imagem_original.png -quality 80 assets/imagem_otimizada.jpg
```

### B. Compressão de Arquivos Grandes
```bash
# Comprimir diretório de assets antigos
tar -czvf jervis/assets_antigos_2025.tar.gz jervis/assets/antigos/
```

### C. Arquivo .gitignore
```bash
# Criar .gitignore
cat > .gitignore << EOF
*.log
*.tmp
*/assets/*.tar.gz
node_modules/
EOF
```

───────────────────────────────────────
🔄 WORKFLOW DE BACKUP PADRÃO:
───────────────────────────────────────

```bash
# 1. Copiar novos arquivos para estrutura
cp /caminho/do/novo_prompt.md jervis/prompts/

# 2. Adicionar mudanças ao staging
git add .

# 3. Criar commit com mensagem descritiva
# (Padrão: Conventional Commits)
git commit -m "feat: Adiciona prompt de engenharia para J-Neuro v1.2"

# 4. Enviar para GitHub
git push origin main
```

───────────────────────────────────────
🤖 AUTOMAÇÃO (Script Shell):
───────────────────────────────────────

**Arquivo: `~/backup_agentes/backup.sh`**

```bash
#!/bin/bash

# Diretório do repositório
REPO_DIR="/home/ubuntu/backup_agentes"
cd $REPO_DIR

# 1. Copiar novos arquivos (ajuste conforme a fonte)
# Exemplo: Encontrar todos os .md modificados nas últimas 24h
# find /caminho/fonte/ -mtime -1 -name "*.md" -exec cp {} jervis/prompts/ \;

# 2. Adicionar, commitar e enviar
git add .
git commit -m "chore: Backup automatizado de $(date +%Y-%m-%d)"
git push origin main

echo "Backup concluído em $(date)"
```

**Tornar executável:**
```bash
chmod +x backup.sh
./backup.sh
```

**Agendar execução diária (01:00):**
```bash
(crontab -l 2>/dev/null; echo "0 1 * * * /home/ubuntu/backup_agentes/backup.sh") | crontab -
```

📊 TABELA RESUMO: AÇÕES X COMANDOS
───────────────────────────────────────

| Ação | Comando | Justificativa |
|------|---------|---------------|
| **Adicionar** | `git add .` | Adiciona mudanças ao staging area |
| **Commit** | `git commit -m "feat: ..."` | Cria ponto de restauração com mensagem padronizada |
| **Push** | `git push origin main` | Sincroniza backup local com GitHub |

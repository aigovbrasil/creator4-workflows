# Creator Operating Suite — Public Demo Video Package

> **Hands-off creator operating suite — video demo package, v1.**
> Pacote completo para gravar, editar, publicar e distribuir a demo pública de 2 minutos do repositório [Creator Operating Suite](https://github.com/Aurelio-Hq/Content-creator-Day-to-day-routine).
> Todos os arquivos são markdown, prontos para copiar para o repositório.

## Índice

| # | Arquivo | O que é | Quando usar |
|---|---|---|---|
| 1 | [`video-concept.md`](./video-concept.md) | Título, promessa, emoção, público, CTA. | Antes de qualquer coisa. Briefing do pacote inteiro. |
| 2 | [`120-second-script.md`](./120-second-script.md) | Roteiro narrado, bloco-a-bloco, com captions on-screen. | Para gravar voz e cronometrar o vídeo. |
| 3 | [`screen-recording-shotlist.md`](./screen-recording-shotlist.md) | 12 cenas: duração, o que mostrar, o que falar, zoom, texto. | Aberto durante a gravação de tela. |
| 4 | [`visual-style-guide.md`](./visual-style-guide.md) | Cores, tipografia, ritmo, captions, lower thirds. | Antes de tocar em qualquer editor. |
| 5 | [`on-screen-copy.md`](./on-screen-copy.md) | Todas as frases que aparecem na tela. | Na edição, para inserir captions. |
| 6 | [`thumbnail-options.md`](./thumbnail-options.md) | 3 variações (direta, provocativa, técnica-traduzida). | Depois da edição, antes de publicar. |
| 7 | [`readme-video-section.md`](./readme-video-section.md) | Bloco markdown para o README do repo. | Após publicar o vídeo. |
| 8 | [`linkedin-demo-post.md`](./linkedin-demo-post.md) | 3 versões de post + follow-ups. | No dia da publicação. |
| 9 | [`medium-demo-section.md`](./medium-demo-section.md) | Trecho para inserir num artigo Medium. | Quando o artigo for publicado. |
| 10 | [`production-checklist.md`](./production-checklist.md) | Checklist linear do zero ao publicado. | Como guia operacional do início ao fim. |

## Ordem sugerida de leitura

1. **`video-concept.md`** — entender o que se está fazendo e por quê.
2. **`120-second-script.md`** — internalizar o roteiro.
3. **`visual-style-guide.md`** — fixar a estética.
4. **`screen-recording-shotlist.md`** + **`on-screen-copy.md`** — guias de produção.
5. **`production-checklist.md`** — executar.
6. **`thumbnail-options.md`** — escolher as 3 thumbs.
7. **`readme-video-section.md`** + **`linkedin-demo-post.md`** + **`medium-demo-section.md`** — distribuir.

## Princípios não-negociáveis

- O vídeo deve ser compreensível **sem áudio**.
- CTA único: **Open the repo, test the starter, and send feedback.**
- Sem "RAG", "embedding", "progressive disclosure", "vector store" no vídeo.
- Sem rosto, sem emoji, sem logo Anthropic/Claude nas thumbs.
- O repo é apresentado como **público em evolução**, nunca como produto final.
